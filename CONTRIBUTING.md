# Contributing to Multi-Agent Decision Engine

First off, thank you for considering contributing to the Multi-Agent platform! It's people like you that make this tool great.

## How Can I Contribute?

### 1. Reporting Bugs
- Ensure the bug was not already reported by searching on GitHub under Issues.
- If you're unable to find an open issue addressing the problem, open a new one. Be sure to include a title and clear description, as much relevant information as possible, and a code sample or an executable test case demonstrating the expected behavior that is not occurring.

### 2. Suggesting Enhancements
- Open a new Issue describing the enhancement. 
- Provide a clear, detailed explanation of the feature and why it would be beneficial to the project.

### 3. Pull Requests
1. **Fork the repo** and create your branch from `main`.
2. **Branch naming convention:** `feature/your-feature-name` or `bugfix/issue-description`.
3. If you've added code that should be tested, add tests.
4. Update the documentation if you change related functionality.
5. Ensure your code passes existing test suites and linting.
6. Issue that pull request!

## Code Style
- **TypeScript**: Follow standard TS style guidelines and the existing ESLint configuration.
- **Python**: We follow `PEP-8`. Please format your code with `black` or `autopep8` before committing.

## Commit Messages
Use the [Conventional Commits](https://www.conventionalcommits.org/) format for all commits. Example: `feat: Add new sales agent logic` or `fix: resolve orchestration timeout`.
